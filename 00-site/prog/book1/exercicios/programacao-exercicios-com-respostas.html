1.1  Repositórios (git)

1.
Após clonar um repositório e modificar seu conteúdo, qual é a sequência de ações a serem realizadas para enviar as atualizações para o repositório remoto?

RESPOSTA:Deve-se adicionar os arquivos a serem incluídos nas modificações; confirmar as alterações (informando uma mensagem); e enviar as alterações para o repositório remoto.

FIM DA RESPOSTA

2.
Suponha que foram modificados diversos arquivos no repositório local, e que deseja-se apenas enviar para o repositório remoto um arquivo chamado “inicio.html”. Tal decisão deve-se ao fato de que se deseja definir uma mensagem de confirmação especial para esse arquivo; as demais modificações poderão ser realizadas com outra mensagem de confirmação das alterações. Forneça o comando completo que informa que apenas o arquivo mencionado será incluído na lista de alterações a ser enviada para o repositório remoto.

RESPOSTA:Obedecendo-se à ordem de comandos utilizados para modificar o repositório remoto (adicionar, confirmar e enviar), o comando que adiciona o arquivo deve ser utilizado da seguinte maneira:

  $ git add inicio.html  
  

FIM DA RESPOSTA

3.
Suponha que você está alterando arquivos de seu repositório. Forneça uma sequência de comandos para enviar as alterações para o repositório remoto.

RESPOSTA:

 $ git add *  
 $ git commit -m ’atualizando’  
 $ git push

FIM DA RESPOSTA

4.
Um aluno de informática geralmente manipula códigos no laboratório da instituição de ensino e em sua casa. Dada uma rotina de trabalho, informe quais são as ações e comandos que o aluno deve fazer quando:

a)
Programa no laboratório de informática.
b)
Programa em casa.

RESPOSTA:

a)
Ao programar no laboratório de informática, o aluno deve:
i.
Se o aluno usa sempre o mesmo computador, pode verificar se já existe o repositório na máquina. Se sim, pode atualizá-lo com git remote update, git status e git pull.
ii.
Se não existir o repositório na máquina, o aluno clona o repositório via git clone (para manter o registro de alterações). Não usar download de arquivo zip, para não perder a característica de repositório.
iii.
Entrar no diretório via terminal (comando cd).
iv.
Após terminar as modificações nos programas, verificar as configurações de user.name e user.email.
v.
Atualizar o repositório remoto com add, commit e push.
b)
Ao programar em casa:
i.
Fazer a comunicação remota via git remote update e git status. Se necessário, atualizar o repositório local (git pull).
ii.
Após modificar os arquivos, fazer a sequência add, commit e push. O usuário já deve estar configurado com a opção --global, que vale para todos os repositórios da máquina.

FIM DA RESPOSTA

7.
O comando ____ ____ ____ ____ ____ remove a configuração global de email do usuário, utilizada pelo git.

RESPOSTA:

git config --global --unset user.email
