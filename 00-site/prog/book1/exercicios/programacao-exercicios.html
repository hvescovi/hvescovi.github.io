6.

Ao executar esse comando, Pedro visualizou como um dos últimos comandos executados o seguinte comando: git config -- global user.email “apsantini@gmail.com”. a)
Pedro pode fazer seus comandos de atualização de dados (add, commit e push), pois o git não vai exibir mensagem de erro, já que há um email configurado de forma global.
b)
Pedro deve remover a configuração global para que não tenha efeito e não haja risco do envio de informação de identificação incorreta para o repositório remoto.
c)
O comando git config --list pode ajudar Pedro a descobrir se a configuração de nome e email estão corretas.
d)
Se houver ambas as configurações definidas (local e global), a configuração global vai prevalecer sobre a configuração local.
